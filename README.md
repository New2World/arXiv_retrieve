# 🌌 ArXiv Agent

**一款自动化 ArXiv 论文抓取与总结系统**
自动抓取 ArXiv 上感兴趣领域的最新学术论文，提取全文并通过大模型总结分析。同时基于用户对论文的阅读喜好不断打分并自我进化。

---

## 🌟 核心特性

### 1. arxiv 论文抓取
支持通过前端界面自由配置关注的学术方向（如 `cs.AI`, `cs.CV`）、回溯时间范围及检索关键词。系统将自动从 ArXiv 检索并获取符合条件的最新论文文献。

### 2. 大模型总结分析
支持自动解析并提取论文 PDF 下载后的完整文本，接入大语言模型（原生支持 OpenAI、Anthropic、Gemini、通义千问、Kimi、智谱 GLM 及本地 Ollama 等多种通道）进行归纳概括，避免只看摘要导致的核心细节遗漏。可以设置并发数与请求间隔以有效防范 API 限流。

### 3. 定时自动抓取
自带稳定的后台时间调度任务系统。用户可以直接在网页后台直观地设定“每天”或“每周”的执行时间点，后台将自动在这个时刻默默为你拉取并生成摘要简报。

### 4. 用户偏好学习
支持在阅读过程中直接为文章进行 1-5 星评分评级，同时支持“不喜欢 (Dislike)” 丢弃流控制。一旦对某篇推送内容标记不感兴趣，系统在下一次抓取循环时会将其从数据库中自动清洗删除，保证不成为后续偏好学习算法的干扰样本。

## 🚀 快速开始

### 方式一：本地部署

**后端：启动 FastAPI 服务**
```bash
cd backend
pip install -r requirements.txt
cp .env.example .env
# [可选] 在 backend/.env 中填入你的大模型 API Key（也可以稍后在前端页面配置）
uvicorn app.main:app --reload --port 8000
```

**前端：启动 React 界面**
```bash
cd frontend
npm install
npm run dev    
# 访问: http://localhost:3000
```

### 方式二：Docker 部署

```bash
cp backend/.env.example backend/.env
# 编辑 backend/.env 并填入 API Key 配置
docker-compose up -d
# 访问: http://localhost:3000
```

## 🧩 界面功能速览

- **🏠 今日论文**：永远只展示最近抓取的论文。
- **📂 历史日报**：按日期回溯每一天生成的文献简报。
- **⚙️ 系统配置**：
  - [检索配置] 随时修改感兴趣的领域(如: `cs.AI`, `cs.CV`)、强关键词检索以及要抓哪几天的数据。
  - [模型配置] 一键切换 LLM 提供商，并在其可用模型间无缝切换，无需重启后端服务。
  - [定时配置] 一键开启自动化定时抓取功能。
